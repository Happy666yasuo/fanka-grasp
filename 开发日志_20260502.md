## MuJoCo Backend Migration

### 新增 MuJoCo simulator

新增 `embodied_agent/src/embodied_agent/mujoco_simulator.py`。

当前实现是 MuJoCo-backed kinematic pick/place simulator：

- 使用 MuJoCo 3.4.0。
- 通过 MJCF string 动态生成 table、blocks、zones。
- 支持 red/blue/yellow blocks。
- 支持 green/blue/yellow zones。
- 使用 kinematic end-effector 与 gripper abstraction。
- 不引入完整 Franka/Panda dynamics 或 IK controller。

该取舍是有意的：当前仓库没有现成 MJCF/Franka 资产，本轮目标是把测试和 CausalExplore simulator-backed 路径迁到 MuJoCo，而不是一次性完成真实机器人动力学建模。

### 新增 simulator protocol 与 factory

新增 `embodied_agent/src/embodied_agent/simulation_protocol.py`，定义 `PickPlaceSimulationProtocol`。

在 `embodied_agent/src/embodied_agent/simulator.py` 中新增 factory：

```python
create_pick_place_simulation(backend="mujoco", **kwargs)
```

支持：

- `backend="mujoco"`：默认新后端。
- `backend="pybullet"` / `backend="bullet"` / `backend="legacy_pybullet"`：保留旧后端。

补充了 backend-neutral helper：

- `get_object_position(object_name)`
- `get_object_pose(object_name)`
- `get_quaternion_from_euler(euler)`

这些接口替代了 Ralph 代码中直接使用 `client_id/object_ids/pybullet.getBasePositionAndOrientation(...)` 的路径。

### PyBullet 改为 legacy 延迟导入

`simulator.py` 中 PyBullet 改为 lazy import。

效果：

- MuJoCo 测试路径不再加载 PyBullet。

## 当前设计判断

### 为什么不是直接接 IsaacLab

项目总目标是 IsaacLab 训练、MuJoCo sim2real 验证。但当前仓库测试和 Ralph 闭环还强依赖 PyBullet 形态的 simulator API。

因此今日采用中间层策略：

1. 先抽象 simulator protocol。
2. 再引入 MuJoCo backend。
3. 让 Ralph 和核心测试默认走 MuJoCo。
4. 后续 IsaacLab env 实现同一 observation/action contract。

这样可以避免把 IsaacLab、MuJoCo、CausalExplore、LLM planner、RL env 一次性耦合在一起。

### 为什么 MuJoCo v1 是 kinematic backend

当前仓库没有现成的 MJCF robot asset，也没有与 IsaacLab 对齐的 robot controller。

因此本轮 MuJoCo backend 聚焦：

- 测试行为等价。
- CausalExplore probe 能在线运行。
- pick/place 状态机与现有 RL env contract 一致。
- 保留后续替换为真实 robot dynamics 的接口空间。

这不是最终 sim2real 物理保真版本，而是迁移主路径和接口边界的 v1。

## 遗留问题

1. MuJoCo backend 仍是 kinematic end-effector，不是完整 Franka/Panda dynamics。
2. 还没有 IsaacLab env 接入同一 protocol。
3. RL 训练仍需要进一步确认是否要从 MuJoCo kinematic env 过渡到 IsaacLab env。
4. Phase3 部分旧 paper/output 材料仍是静态产物，虽然已更新主环境口径，但实验数值本身还应在正式论文前重新跑一版。
5. 当前目录没有 `.git`，无法用 git diff/status 做变更归档。
6. 测试中仍有 Matplotlib cache warning 和 Gym deprecated warning，暂未处理，因为不影响本轮目标。

## 下一步建议

1. 定义 IsaacLab env 的最小 `PickPlaceSimulationProtocol` adapter。
2. 为 MuJoCo backend 增加更真实的 robot asset/controller，或明确它只作为 sim2real validation scene。
3. 把 Phase3 实验重新跑一版，生成与 MuJoCo 口径一致的新 comparative/ablation report。
4. 继续清理核心代码中对 legacy PyBullet 类型名的少量历史痕迹，只保留在 legacy backend 文件或兼容层中。
5. 为 `create_pick_place_simulation` 增加 config/env var backend 选择，例如 `EMBODIED_SIM_BACKEND=mujoco|pybullet|isaaclab`。

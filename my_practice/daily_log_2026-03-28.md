### 5. 全面添加显存约束（v2.1 升级）

根据本机显存有限的实际情况，**对所有 6 个文件统一注入显存保护规则**：

| 规则 | 具体约束 |
|------|---------|
| `--headless` | **所有** IsaacSim/IsaacLab 命令必须携带，GUI 渲染会耗尽显存 |
| `num_envs` | 冒烟测试 ≤ 4，短验证 ≤ 16，正式训练 64~128（绝不盲目开 256+） |
| `buffer_size` | 默认 `100_000`（非 1M），避免 CPU/GPU 内存溢出 |
| OOM 应对 | 优先降 `num_envs` → 其次降 `batch_size` → 最后降 `buffer_size` |

## 💡 经验与收获

1. **Agent Skills 标准**：Anthropic 发起的开放标准，`.agents/skills/<name>/SKILL.md` 格式可被多种 AI 编辑器自动发现，比手动粘贴上下文高效得多
2. **AGENTS.md 全局规则**：放在项目根目录，Copilot/Cursor/Windsurf 会自动加载，适合存放安全规则和环境约束
3. **显存保护必须前置**：`--headless` 和 `num_envs` 限制如果不写进文档，AI 很容易默认开 256+ 环境导致 OOM
4. **文档即协议**：prompts.md + verify_checklist.md + checkpoint.md 构成了"AI 自动执行 → 自动验证 → 自动记录"的闭环
5. **Git SSH deploy key 有仓库限制**：推送新仓库时 SSH key 可能没权限，切 HTTPS 即可解决

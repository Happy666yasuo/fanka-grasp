### 2. 纯强化学习（RL）训练 ✅

由于大创项目主题为"基于强化学习"，BC 是监督学习并不符合核心声明，因此新建 `project_RL/` 目录进行纯 RL 实验。

#### 2.1 环境设计：DensePickEnv

文件：`project_RL/dense_pick_env.py`

- **观测空间**（17维）：末端执行器位置、物体位置、目标区域位置、相对向量、夹爪开合度、持握标志
- **动作空间**（4维连续）：`[Δx, Δy, Δz, gripper_command]`，范围 [-1, 1]
- **密集奖励函数经历了 3 次迭代**：

| 版本 | 问题 | 修复 |
|------|------|------|
| **v1** | 接近奖励(~29/ep) > 成功奖励(~20/ep)，智能体学会原地不动 | 添加 -0.5 时间惩罚基线 |
| **v2** | hold=10/步 × 20步 = 200 >> success=50，智能体学会抓住但不抬起 | — |
| **v3（最终版）** | ✅ hold 20步=40 vs 成功3步=218，正确激励快速完成 | hold=2.0, lift=100×h, success=+200 |

**最终奖励函数 (v3)**：
```
未持握: -0.5 + 0.5*(1-tanh(5*dist)) + gripper_close_bonus
持握中: 2.0 + 100.0 * lift_height
成功:   +200.0 终止奖励
```

## 三、关键经验与教训

1. **奖励设计是 RL 的核心难点**：三次迭代才找到正确的奖励平衡。核心原则是"成功的总回报必须远大于任何非成功行为的累积回报"。
2. **SAC 自适应熵不稳定**：在小环境中，自适应熵调整会导致策略崩溃（100% → 0%）。固定 `ent_coef=0.01` 后立刻稳定。
3. **早停非常重要**：RL 训练可能在短时间内达到最优然后退化，必须在高成功率时保存并停止。
4. **2000 步即可收敛**：得益于良好的奖励设计 + SAC 的样本效率 + staging pose 初始化，仅需 108 秒即可训练出 100% 成功率的策略。

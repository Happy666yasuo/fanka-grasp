## 今日已完成任务归档

### 1. P0 总计划与基线冻结

- 创建 `NewPlan.md`，把三人团队后续主线统一成“大脑 LLM planner + CausalExplore 认知桥梁 + 小脑技能执行”的总蓝图。
- 创建 `P0_BASELINE_FREEZE.md`，明确 staged chain、task-pool place、recovery telemetry 的当前接受基线。
- 创建 `SYSTEM_INTERFACES.md`，固定 CausalExplore、Planner、Executor 的统一 schema。
- 创建 `P0_ACCEPTANCE_CHECKLIST.md`，把 P0 验收项显式化。

### 2. P1 结构化接口与 planner export

- 新增 contract dataclass，统一 `CausalExploreOutput / PlannerStep / ExecutorResult / FailureRecord / ContactState`。
- 新增 planner-facing adapter，把现有 `RuleBasedPlanner` 的输出导成结构化 contract。
- 新增 planning bridge，在高不确定对象上自动插入 `probe`，并覆盖 `plan_contract` 与 `replan_contract` 两条路径。

### 3. P1 raw place runtime alignment 准备链

- 为 place runtime reset dataset builder 增加 `broadened_v1` preset。
- 生成 `task_pool_post_pick_broadened_v1` 候选数据集与审计文件。
- 新增对应训练配置 `place_sac_runtime_aligned_bcinit_task_pool_broadened_v1.yaml`。
- 补齐 raw place 诊断计划、artifact review、support comparison、dataset spec 和临时审计文档。

### 4. P2 artifact-backed mock integration

- 创建 repo 内固定 fixture：`mock_registry_v1`，包含 `registry.json`、`red_block.json`、`blue_block.json`。
- 新增 `run_mock_planning_bridge.py`，可直接输出 planning-only JSON。
- 新增 `run_mock_system_demo.py`，可直接输出 `causal_outputs / planner_steps / executor_result / replanned_steps`。
- 增加标准 named scenario：
  - `uncertain_red_to_green`
  - `confident_blue_to_yellow`
  - `blue_success`
  - `red_place_failure`
- 新建 `P2_MOCK_INTEGRATION_STATUS.md`，明确当前只是 artifact-backed mock integration，不是 simulator-backed CausalExplore 真闭环。

### 5. 真实 provider 适配层第一版

- 在 `planning_bridge.py` 新增 `SimulatorArtifactCatalogCausalOutputProvider`。
- 支持从 simulator-style catalog 读取：
  - `object_manifest_path`
  - object-level `artifact_path`
  - artifact 内 `scene_id`
- 在 planning runner 和 system runner 中加入 `catalog_path` 入口，允许未来真实 simulator-backed artifact 直接复用当前 contract 链路。
